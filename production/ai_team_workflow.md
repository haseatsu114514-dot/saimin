```

│                 SHIZM AI プロダクションチーム                   │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  【ライター / オペレーター】Claude                              │

│  ├─ 構成設計・4フェーズの時間配分                              │
│  ├─ 安全性チェック・分析・改善提案                             │
│  └─ PC操作の自動化（YouTube API・ファイル管理・SNS投稿）        │
│                                                              │
│  【企画・SEO・SNS】ChatGPT                                   │
│  ├─ テーマのブレスト・壁打ち・Web検索                          │
│  └─ タイトル・説明文・タグ・チャプター・SNS投稿文の生成          │
│                                                              │
│  【アートディレクター】ナナバナ（Gemini）                       │
│  └─ サムネ・背景画像・アイコン・バナーの生成                    │
│                                                              │
│  【サウンドエンジニア】Adobe Podcast                           │
│  └─ AIノイズ除去・ラウドネス調整                               │
│                                                              │
│  【パフォーマー / 最終決定者】自分                              │
│  ├─ 台本の最終調整・読み上げ収録                               │
│  └─ 各工程の最終チェック・GO判断                               │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

### Claude vs ChatGPT の使い分け

```
  Claude                            ChatGPT
  ─────────────────────             ─────────────────────
  台本を書く           ◀──────▶     アイデアを広げる
  構成を設計する       ◀──────▶     SEO・メタデータを作る
  安全性を確認する     ◀──────▶     SNS投稿文を作る
  分析・改善する       ◀──────▶     壁打ち・Web検索
  PC操作を自動実行する

  【原則】
  ・「台本を書く」→ Claude（リポジトリ参照）
  ・「それ以外のテキスト」→ ChatGPT
  ・「実行する」→ Claude（API操作・ファイル管理）
  ・迷ったら両方に投げて良いほうを採用
```

### 集計

```
全10工程の内訳:
  AI がやる    ─── 5工程（工程 1, 2, 4, 9, 10）
  自分がやる   ─── 2工程（工程 3, 6）
  両方        ─── 3工程（工程 5, 7, 8）

自分が必ずやること:
  ・台本を声に出して読む → 修正する（3）
  ・音声を収録する（6）
```

## 自動化の方針

### Claude による PC操作の自動化

```
  ◎ Claude が自動実行するもの:
    ・YouTube アップロード       → YouTube Data API v3 経由
    ・メタデータ設定             → タイトル・説明文・タグ・サムネイル
    ・公開/予約公開              → API で公開設定
    ・X 投稿                    → X API 経由で自動投稿
    ・ファイル管理               → 書き出し済み動画のリネーム・整理
    ・台本のリポジトリ保存        → git 操作

  ※ 初回セットアップが必要:
    ・Google Cloud Console で YouTube Data API の OAuth 認証
    ・X Developer Portal で API キー取得
    ・認証情報を安全に保管（.env 等）

  × やらない:
    ・自動いいね/自動フォロー
      → X・YouTube ともに規約違反でアカウント凍結リスク
      → 催眠チャンネルの「信頼感」を損なう

  △ 手動で行う:
    ・関連チャンネルへのコメント等の自然な交流
```

### 自動化できない工程（GUI操作が必要なもの）

```
  以下はアプリのGUI操作が必要なため、自動化が難しい:
    ・GarageBand での EQ/コンプ/リバーブ調整
    ・iMovie/DaVinci での動画書き出し
    ・Canva でのテキスト配置
    → プリセット化・テンプレ化で省力化するのが現実的
```
